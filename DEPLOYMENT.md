# GitHub Integration Setup Guide

## Prerequisites
- GitHub repository created
- Railway account (https://railway.app)
- Vercel account (https://vercel.com)
- Both accounts connected to your GitHub

## Step 1: Set Up GitHub Secrets

Add these secrets to your GitHub repository (Settings → Secrets and variables → Actions):

### Railway Secrets
- `RAILWAY_TOKEN` - Get from Railway dashboard (Account → API Tokens)
- `RAILWAY_PROJECT_ID` - Your Railway project ID (optional, can be set in railway.json)

### Vercel Secrets
- `VERCEL_TOKEN` - Get from Vercel account settings
- `VERCEL_ORG_ID` - Your Vercel organization ID
- `VERCEL_PROJECT_ID` - Your Vercel project ID

## Step 2: Connect Railway

1. Go to https://railway.app and create a new project
2. Connect your GitHub repository
3. Create two services:
   - **Backend**: Deploy from `backend/` directory
   - **PostgreSQL**: Add PostgreSQL plugin
   - **Redis**: Add Redis plugin (optional but recommended)

4. Set environment variables in Railway:
   ```
   DATABASE_URL = [auto-generated by PostgreSQL plugin]
   REDIS_URL = [auto-generated by Redis plugin]
   FRONTEND_URL = [your Vercel domain]
   JWT_SECRET = [generate a secure key]
   SERP_API_KEY = [your SERP API key]
   NODE_ENV = production
   ```

5. Copy your Railway domain URL to use as `VITE_API_URL` in Vercel

## Step 3: Connect Vercel

1. Go to https://vercel.com and create a new project
2. Select "Other" and link your GitHub repository
3. Configure project:
   - **Root Directory**: `frontend`
   - **Build Command**: `npm run build`
   - **Output Directory**: `dist`

4. Add environment variables:
   ```
   VITE_API_URL = https://your-railway-backend-url.com
   ```

## Step 4: Update GitHub Workflow

The `.github/workflows/deploy.yml` is configured to:
- Run tests on every push and PR
- Auto-deploy to Railway (backend) on push to main
- Auto-deploy to Vercel (frontend) on push to main

## Step 5: First Deployment

1. Push a commit to the `main` branch
2. GitHub Actions will:
   - ✅ Run tests
   - ✅ Build both backend and frontend
   - ✅ Deploy backend to Railway
   - ✅ Deploy frontend to Vercel
3. Monitor progress in GitHub Actions tab
4. Check Railway and Vercel dashboards for deployment status

## Database Migrations

For running database migrations on Railway:
1. Go to Railway dashboard → Your Backend Service
2. Open the "Deploy" tab
3. Add this to your pre-deployment script:
   ```bash
   npm run migrate
   ```

## Environment Variables Reference

### Backend (Railway)
- `DATABASE_URL`: PostgreSQL connection string (auto-set)
- `REDIS_URL`: Redis connection string (auto-set)
- `JWT_SECRET`: Secret for JWT tokens
- `FRONTEND_URL`: Your Vercel domain
- `NODE_ENV`: Set to `production`
- `PORT`: Usually auto-set to 3000
- `SERP_API_KEY`: Your SERP provider API key

### Frontend (Vercel)
- `VITE_API_URL`: Your Railway backend domain

## Monitoring & Debugging

### Railway Logs
- Railway Dashboard → Your Service → Logs tab

### Vercel Logs
- Vercel Dashboard → Your Project → Deployments → Click deployment → Logs

### GitHub Actions
- GitHub Repository → Actions tab

## Common Issues

### Build Failures on Railway
- Check that `npm run build` works locally
- Ensure `Dockerfile` is present in `backend/` directory
- Verify all dependencies are in `package.json`

### Frontend Can't Connect to Backend
- Ensure `VITE_API_URL` is set correctly in Vercel
- Check CORS is enabled in backend (`corsOptions` in server.ts)
- Verify backend is running (check Railway logs)

### Database Connection Errors
- Verify `DATABASE_URL` is set in Railway
- Check that PostgreSQL plugin is running
- Run migrations if schema is missing
